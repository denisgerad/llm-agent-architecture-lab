# LLM vs Agent Pipeline — Mock Demo
### Fully deterministic, offline pipeline

A self-contained demo of the full LLM → Agent pipeline. All outputs are generated by deterministic mock functions — **no AI server, no API keys, no internet connection required.**

---

## 📋 Stages

| File | Stage | What it shows |
|---|---|---|
| `stage1-llm-vs-agent-design-demo.html` | Stage 1 | LLM produces a structured design spec (project overview, modules, design decisions) |
| `stage2-planning-eslint.html` | Stage 2 | LLM translates design decisions into ESLint / Prettier / Stylelint config files |
| `stage25-blueprint.html` | Stage 2.5 | LLM produces a full implementation blueprint — folder tree, file manifest, module mapping, agent package |
| `stage3-developer-discipline.html` | Stage 3 | Developer discipline guide — rules and constraints the agent follows during scaffolding |

---

## 🚀 Running

Open any file directly in a browser — no server needed:

```
mock-demo/stage1-llm-vs-agent-design-demo.html
```

Or serve locally if you prefer:

```powershell
python -m http.server 8000
# then open http://localhost:8000
```

---

## 🧠 Why Mock-Only?

- **Instant** — no waiting for an LLM to respond
- **Consistent** — same output every time, good for presentations and screenshots
- **Offline** — works without Ollama, OpenAI, or any network access
- **Predictable** — outputs are realistic but deterministic, so you always know what to expect

> For the version with real Ollama + Mistral AI enrichment, see the `../llm-demo/` folder.

---

## 🔄 Pipeline Flow

```
Stage 1: Design (LLM)
  → Structured project spec: modules, tech decisions, tradeoffs

Stage 2: Rules (LLM)
  → ESLint / Prettier / Stylelint configs derived from design decisions

Stage 2.5: Blueprint (LLM)
  → Folder structure, file manifest, module mapping, agent execution package

Stage 3: Developer Discipline
  → Constraints and rules passed to the Stage 3 scaffolding agent
```

The output of each stage feeds into the next. Stage 2.5 produces the `agent-execution-package.json` that the Stage 3 agent uses to scaffold the project with zero assumptions.
